# /analyze 스킬

복잡한 분석 작업을 서브에이전트에서 처리하여 메인 컨텍스트 오염을 방지합니다.

## 사용법

```
/analyze [분석 대상 및 질문]
```

## 예시

```
/analyze 번역기 성능 병목 지점
/analyze SSG 빌드 최적화 가능성
/analyze 타입 안전성 개선점
/analyze 코드 중복 패턴
```

## 실행 규칙

1. Task tool 호출
2. `subagent_type: "Plan"` 또는 `"Explore"` (분석 유형에 따라)
3. `thoroughness: "very thorough"`
4. 결과를 구조화된 형태로 반환 요청

## 프롬프트 템플릿

```
분석 요청: {user_query}

다음을 수행하세요:
1. 관련 코드 전체 탐색
2. 심층 분석 수행
3. 문제점/개선점 도출
4. 우선순위 제안

결과는 다음 형식으로 반환:
- 현재 상태 요약
- 발견된 문제점/개선점 (우선순위 포함)
- 구체적 액션 아이템
```

## 적합한 작업

- 성능 분석
- 아키텍처 리뷰
- 리팩토링 계획
- 보안 검토
- 코드 품질 분석

## 부적합한 작업

- 단순 검색 (→ /search)
- 구조 파악 (→ /explore)

## 결과 형식

```markdown
## 현재 상태
[요약]

## 발견 사항
| 우선순위 | 항목 | 설명 |
|---------|------|------|
| 높음 | ... | ... |
| 중간 | ... | ... |
| 낮음 | ... | ... |

## 액션 아이템
1. ...
2. ...
```

## 효과

- 메인 컨텍스트 오염 방지
- 토큰 절약: 약 60-70%
- 체계적인 분석 결과
